===============================================================================

METHOD A: Using Cursor's Command Palette (Recommended)
-------------------------------------------------------
1. Open Cursor
2. Press: Cmd+Shift+P (Mac) or Ctrl+Shift+P (Windows/Linux)
3. Type: "Remote-SSH: Connect to Host"
4. Select: big-data-bowl
5. Cursor will connect and reload in a new window

METHOD B: Using SSH Remote Menu
--------------------------------
1. Look at the bottom-left corner of Cursor
2. Click the green/blue remote connection button (><)
3. Select "Connect to Host"
4. Choose: big-data-bowl

===============================================================================
TROUBLESHOOTING
===============================================================================

ISSUE: "Connection refused" or "Host key verification failed"
SOLUTION:
  ssh-keygen -R 13.56.165.161
  Then reconnect

ISSUE: Cursor can't find the host
SOLUTION:
  1. Check SSH config: cat ~/.ssh/config
  2. Test connection: ssh big-data-bowl
  3. Make sure instance is running:
     aws ec2 describe-instances --instance-ids i-0ee7647bb88ef1f71

ISSUE: IP address changed (after stopping/starting instance)
SOLUTION:
  1. Get new IP:
     NEW_IP=$(aws ec2 describe-instances --instance-ids i-0ee7647bb88ef1f71 \
         --query 'Reservations[0].Instances[0].PublicIpAddress' --output text)
  
  2. Update SSH config:
     sed -i.bak "s/HostName .*/HostName $NEW_IP/" ~/.ssh/config
  
  3. Reconnect from Cursor

ISSUE: Extensions not working
SOLUTION:
  • Some Cursor extensions need to be installed on the remote server
  • Cursor will prompt you - click "Install in SSH: big-data-bowl"
